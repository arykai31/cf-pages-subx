# 项目目录结构

本文档描述了项目的整体目录架构，采用分层设计模式，包含表现层、应用层、领域层、共享层和配置层。

---

## 1. 源代码目录（src）

源代码目录包含所有业务逻辑和用户界面代码，按功能分层组织。

### 1.1 表现层（presentation）

负责用户界面的渲染和交互，包含通用UI组件、页面组件、全局样式和UI专用hooks。

```
src/presentation/
├── components/        # 通用UI组件
│   ├── atoms/         # 原子组件
│   ├── molecules/     # 分子组件
│   ├── organisms/     # 有机体组件
│   └── layouts/       # 布局组件
├── pages/             # 页面组件
├── styles/            # 全局样式
└── hooks/             # UI专用hooks
```

### 1.2 应用层（application）

处理业务逻辑和业务流程，协调领域层和表现层之间的交互。

```
src/application/
├── hooks/             # 业务逻辑hooks
├── services/          # 应用服务
├── stores/            # 状态管理
└── workflows/         # 业务流程
```

### 1.3 领域层（domain）

包含核心业务逻辑和规则，是项目的核心部分。

```
src/domain/
├── entities/          # 领域实体
├── value-objects/     # 值对象
├── repositories/      # 仓储接口
├── services/          # 领域服务
└── events/            # 领域事件
```

---

## 2. 共享层（shared）

包含整个项目中可复用的类型定义、工具函数、常量和验证逻辑。

```
shared/
├── types/             # 类型定义
├── utils/             # 工具函数
├── constants/         # 常量
└── validation/        # 验证逻辑
```

---

## 3. 配置层（config）

包含项目的各种配置文件。

```
config/
├── env.ts             # 环境配置
├── routes.ts          # 路由配置
├── theme.ts           # 主题配置
└── features.ts        # 特性开关配置
```

---

## 4. 目录结构说明

本项目采用分层架构设计，各层职责清晰：

- **表现层（presentation）**：用户界面和交互
- **应用层（application）**：业务流程编排
- **领域层（domain）**：核心业务逻辑
- **共享层（shared）**：通用组件和工具
- **配置层（config）**：项目配置管理

---

## 5. 设计原则

1. **高内聚低耦合**：每个目录和文件职责单一
2. **分层清晰**：表现层、应用层、领域层分离
3. **易于维护**：模块化设计，便于扩展和维护
4. **可复用性**：共享层组件可在多个项目中复用
