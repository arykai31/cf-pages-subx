## 前端架构设计

### 技术栈确认

- 前端框架：React 18
- 状态管理：Zustand
- 路由：React Router DOM
- 样式：Tailwind CSS
- 组件库：Shadcn/UI
- 类型检查：TypeScript
- 代码格式化：Prettier + ESLint
- 构建工具：Vite
- HTTP 客户端：Fetch API
- 图标：Heroicons, lucide-animated, lucide-react
- 部署平台：Cloudflare Pages

### 分层策略

依赖方向：依赖应该从上层指向下层，绝不反向
    表现层 → 应用层 → 领域层 ← 基础设施层
    共享层可被任何层使用，但不应依赖其他层
接口隔离：
    领域层定义接口
    基础设施层实现接口
    应用层使用接口，不关心具体实现
单一职责原则：
    每层只负责自己的职责
    避免跨层调用（如表现层直接调用基础设施）
测试策略：
    表现层：组件测试、快照测试
    应用层：集成测试
    领域层：单元测试
    基础设施层：端到端测试、mock测试
演进式架构：
    从小型项目架构开始
    随着复杂度增加，逐步引入更严格的分层
    避免过度工程化：
        只在必要时引入新的层
        保持简单而不是复杂
